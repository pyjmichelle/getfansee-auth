# 飞书文档 Skill

当用户提供飞书（Feishu / Lark）文档内容时，将其视为需求或规范的主要依据。

## 使用时机

- 用户说「给你飞书的文档」「按飞书文档做」「参考这份飞书」
- 用户粘贴来自飞书文档的文本、表格或列表
- 用户分享飞书文档链接（如 `https://xxx.feishu.cn/docx/...`）
- 用户提供从飞书导出的 Markdown / 文档内容

## 处理方式

### 1. 用户直接粘贴内容

- 将粘贴的飞书内容视为**规范或需求**的权威来源。
- 若包含标题层级、列表、表格，按原有结构理解（章节、子项、字段含义）。
- 实现或审查代码时，以文档中的描述为准；有歧义时优先按文档表述执行，必要时向用户确认。

### 2. 用户提供飞书链接

- 无法直接访问飞书链接内容时，提示用户：
  - 复制文档正文粘贴到对话中，或
  - 从飞书导出为 Markdown/Word 后粘贴或上传。
- 若用户已粘贴对应内容，则按「直接粘贴内容」方式处理。

### 3. 文档结构约定

- **多级标题**：视为需求/规范的章节与子章节。
- **表格**：表头为字段/配置名，行为主要内容；可用于接口字段、配置项、用例等。
- **列表/待办**：视为步骤、检查项或功能点列表。
- **加粗/高亮**：视为重点或必选项，实现时优先满足。

## 工作流

1. **确认来源**：识别用户提供的是飞书文档内容或导出内容。
2. **提取要点**：从标题、列表、表格中提取需求、字段、规则、验收条件。
3. **对齐实现**：编码、改代码或写测试时，以文档为准并标注对应位置（如「按飞书文档 3.2 节」）。
4. **歧义处理**：文档与现有实现或常识冲突时，指出冲突并请用户确认以哪边为准。

## 输出习惯

- 引用文档时注明来源，例如：「根据飞书文档【xxx】章节」。
- 若把文档整理成任务列表或规范摘要，可先给用户确认再继续实现。
