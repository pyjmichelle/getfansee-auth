# GetFanSee - UI 设计需求清单

> 给 UI 设计师的完整功能需求文档，需要同时设计 Mobile 和 PC/Desktop 两套界面。

**像素级实现规范**：参见 [FIGMA_PIXEL_SPEC.md](../design/FIGMA_PIXEL_SPEC.md)，所有实现必须遵循该规范中的间距、字号与组件尺寸。

---

## 产品简介

**GetFanSee** 是一个创作者经济平台，让创作者可以发布付费内容，粉丝可以订阅或单次购买来解锁内容。

**两种用户角色**：

- **Fan（粉丝）**：浏览内容、订阅创作者、购买付费内容、互动
- **Creator（创作者）**：发布内容、设置价格、查看数据、管理收益

## 一、认证模块

### 1.1 登录页面

**Mobile + PC**

需要元素：

- 产品 Logo 和名称
- 邮箱输入框
- 密码输入框（支持显示/隐藏密码）
- 登录按钮
- "忘记密码" 链接
- 切换到注册的入口
- （可选）第三方登录按钮（Google）

### 1.2 注册页面

**Mobile + PC**

需要元素：

- 邮箱输入框
- 密码输入框（提示：至少8位）
- 确认密码输入框
- 18岁以上确认勾选框（必选）
- 同意服务条款勾选框
- 注册按钮
- 切换到登录的入口

### 1.3 邮箱验证页面

**Mobile + PC**

需要状态：

- 验证中（加载状态）
- 验证成功（成功提示，自动跳转）
- 验证失败（错误提示 + 重新发送按钮）

### 1.4 重新发送验证邮件页面

**Mobile + PC**

需要元素：

- 邮箱输入框
- 发送按钮
- 成功/失败提示

## 二、内容浏览模块

### 2.1 首页 Feed

**Mobile + PC**

功能需求：

- 内容卡片流（瀑布流或列表流）
- 下拉刷新
- 无限滚动加载更多
- 空状态（没有内容时）

**内容卡片需要包含**：

- 创作者头像、名称、用户名
- 发布时间
- 帖子文字内容
- 媒体内容区域（图片/视频）
- 锁定状态显示（如果是付费内容）
- 互动栏：点赞数、评论数、分享、打赏

**内容锁定状态**：

- 订阅者专享内容：模糊遮罩 + "订阅解锁" 按钮
- PPV 付费内容：模糊遮罩 + 价格 + "购买解锁" 按钮

### 2.2 帖子详情页

**Mobile + PC**

需要元素：

- 返回按钮
- 创作者信息（头像、名称、关注按钮）
- 完整帖子内容
- 媒体内容（支持图片轮播、视频播放）
- 锁定遮罩（如果未解锁）
- 点赞、分享、举报按钮
- 评论区：
  - 评论输入框
  - 评论列表（头像、用户名、内容、时间、删除按钮）
  - 加载更多评论

### 2.3 创作者主页

**Mobile + PC**

需要元素：

- 封面图区域
- 头像
- 创作者名称、用户名
- 简介/Bio
- 统计数据（帖子数、订阅者数）
- 订阅按钮（显示价格）/ 已订阅状态
- 内容标签页切换（全部帖子 / 媒体 / 已点赞）
- 帖子列表/网格

### 2.4 搜索页面

**Mobile + PC**

需要元素：

- 搜索输入框
- 搜索类型切换（创作者 / 帖子 / 标签）
- 搜索结果列表
- 热门搜索/推荐（可选）
- 空结果状态
- 搜索历史（可选）

### 2.5 标签页面

**Mobile + PC**

需要元素：

- 标签名称和描述
- 该标签下的帖子列表
- 帖子数量统计

## 四、创作者工具模块

需要元素：

- 核心数据卡片：
  - 总收入
  - 订阅者数量
  - PPV 销售数
  - 内容浏览量

**Mobile + PC**

- 返回按钮

  - 缩略图

## 九、关键用户流程

请确保以下完整流程的设计连贯：

### 流程 1：新用户注册到浏览内容

```
注册 → 邮箱验证 → 首页 Feed → 浏览帖子
```

### 流程 2：粉丝订阅创作者

```
浏览帖子 → 看到锁定内容 → 点击订阅 → 订阅弹窗 → 余额不足则充值 → 订阅成功 → 解锁内容
```

### 流程 3：粉丝购买 PPV 内容

```
浏览帖子 → 看到 PPV 内容 → 点击解锁 → 解锁弹窗 → 确认购买 → 解锁成功
```

### 流程 4：粉丝升级为创作者

```
个人资料页 → 点击升级 → 填写资料 → 提交 KYC → 等待审核 → 审核通过 → 进入工作室
```

### 流程 5：创作者发布内容

```
工作室 → 新建帖子 → 填写内容 → 上传媒体 → 设置定价 → 发布 → 成功页面
```

### 流程 6：创作者查看收益

```
工作室 → 收入管理 → 查看明细 → （提现）
```

## 十、设计交付要求

### 需要交付的设计稿

1. **Mobile 版本**（以 375px 或 390px 宽度为基准）
   - 所有页面的移动端设计
   - 底部导航栏
   - 抽屉式菜单

2. **PC/Desktop 版本**（以 1440px 或 1920px 宽度为基准）
   - 所有页面的桌面端设计
   - 顶部导航栏
   - 适当的多列布局

3. **组件库**
   - 按钮（各种状态：默认、悬停、按下、禁用、加载中）
   - 输入框（各种状态）
   - 卡片
   - 弹窗
   - 导航组件
   - 状态组件

4. **交互说明**
   - 关键交互的动效说明
   - 页面跳转逻辑

### 设计原则建议

- 需要同时支持亮色和暗色模式
- 注意触摸友好（移动端按钮和可点击区域足够大）
- 考虑各种状态（加载、空、错误、成功）
- 付费内容的锁定状态要明显但不影响浏览体验
